## Purpose
Shared test utilities: API runner, auth token management, DB assertions, route resolution.

## Key Functions

### `api(method, path, options)`
**File**: `api.ts`
**Purpose**: Generic API runner via `testApiHeader`. Adds Bearer token, JSON body, query params.
**Used by**: `tests/api/` — API route unit tests

### `makeRequest(method, path, options)`
**File**: `request.ts`
**Purpose**: Same as `api()` but for integration tests. Auto-sets Content-Type.
**Used by**: `tests/integration/` — Integration tests

### Auth helpers
**File**: `auth.ts`
**Exports**:
- `getOwnerToken()` → Signs in owner user, caches access token
- `getPhotographerToken()` → Photographer user token
- `getEditorToken()` → Editor user token
- `getOutsiderToken()` → User from different studio (for RLS tests)
- All use password: `Test@1234`

### DB helpers
**File**: `db.ts`
**Exports**:
- `getRow(table, id)` → Fetches row via Supabase admin
- `getCount(table, filter)` → Returns count via Supabase admin
- `getRaw(sql)` → Direct PostgreSQL query via `pg` Client

### Token helpers
**File**: `tokens.ts`
**Exports**: `setupTokens()`, `tokens` object (raw string tokens for deep tests)

### Route resolution
**File**: `resolve-route.ts`
**Exports**: `loadAppRoute(pathname)` → Dynamic `import()` of Next.js route handler
**Purpose**: Maps ~90 URL patterns to their route.ts file for `testApiHeader`
