### Clients, Leads, Invoices, Galleries, Team, Packages, Contracts, Proposals
All follow standard CRUD pattern:
- **GET** list → `requireAuth`, paginated response
- **GET** detail → `requireAuth`, single object response
- **POST** create → `requireAuth` or `requireOwner`, 201 response
- **PATCH** update → `requireOwner` (except bookings: `requireAuth`)
- **DELETE** → `requireOwner`, 204 response

## Common Patterns
- **Auth guards**: `requireAuth` for reads, `requireOwner` for writes
- **Response format**: `Response.ok/created/noContent/paginated/error` from `../../lib/.ai.md`
- **Error handling**: Manual try/catch with `ServiceError` or `handleRouteError()`
- **Validation**: Zod `.safeParse()` at route or service level
- **Caching**: Dashboard/analytics use `withCache()`, mutations use `withNoStore()`
