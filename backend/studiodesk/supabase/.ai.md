# Supabase Module

## Purpose
Database configuration, migration management, and seed scripts for the StudioDesk Supabase backend. Contains seeding utilities for test data, admin users, and database verification tools.

## Configuration

### config.toml
Supabase CLI configuration. Defines project settings, database connection, and edge function configuration.

### .gitignore
Excludes sensitive/generated files from version control.

## Seed Scripts

### seed.ts
Main comprehensive seed script for local development (Modules 1 & 2).
- **Purpose**: Populates database with 2 studios, 4 users, packages, clients, leads, bookings, contracts, invoices, payments, assignments, galleries, automation data, revenue snapshots, and onboarding events.
- **Usage**: `npm run db:seed` (runs `npx ts-node -r tsconfig-paths/register --project supabase/tsconfig.seed.json supabase/seed.ts`)
- **Async**: Yes
- **Key Operations**:
  1. Creates auth users (owner, photographer, editor, outsider) with idempotency
  2. Creates studios (XYZ Photography - Studio A, ABC Clicks - Studio B)
  3. Creates onboarding events for Studio A
  4. Creates subscription plans (Starter, Studio, Agency)
  5. Creates studio members with roles
  6. Creates service packages (4) and addons (3)
  7. Creates clients (6 across 2 studios)
  8. Creates leads (8), bookings (9), contracts (4), invoices (6)
  9. Creates payments, assignments, galleries, face clusters
  10. Seeds automation settings, WhatsApp/email templates, automation logs
  11. Seeds 90 days of revenue snapshots
- **Deterministic UUIDs**: All entities use predefined IDs from `seed-ids.ts`
- **Password**: `Test@1234` for all test users
- **Calls To**: `./seed-ids.ts`, `@/lib/crypto` (encrypt)
- **Environment**: Requires `SUPABASE_SERVICE_ROLE_KEY`, `NEXT_PUBLIC_SUPABASE_URL`

### seed-super-admin.ts
Creates the platform super admin user.
- **Purpose**: Seeds a single `super_admin` platform admin for accessing the admin panel.
- **Usage**: `npx ts-node -r tsconfig-paths/register --project supabase/tsconfig.seed.json supabase/seed-super-admin.ts`
- **Async**: Yes
- **Credentials**:
  - Email: `admin@studiodesk.in`
  - Password: `Admin@1234`
  - Role: `super_admin`
- **Idempotency**: Checks existing auth users and platform_admins records before creation
- **Operations**:
  1. Creates/verifies auth user via Admin API
  2. Creates/updates `platform_admins` record
- **Deterministic UUID**: `d0000001-0001-4000-8000-000000000001`

### seed-test-users.ts
Standalone script to create 4 test users and their studio member records.
- **Purpose**: Subset of main seed focusing only on auth users and studio_members.
- **Usage**: `npm run db:seed:users`
- **Users Created**:
  - `owner@test.com` → OWNER, Studio A (XYZ Photography)
  - `photographer@test.com` → PHOTOGRAPHER, Studio A
  - `editor@test.com` → EDITOR, Studio A
  - `outsider@test.com` → OWNER, Studio B (ABC Clicks)
- **Password**: `Test@1234` (hashed by Supabase GoTrue/bcrypt)
- **Idempotency**: Checks auth users before creation, upserts studio_members on conflict
- **Verification**: Runs post-seed verification queries
- **Calls To**: Deterministic UUIDs (inline, same as seed-ids.ts)

### rollback-test-users.ts
Rollback/delete test users.
- **Purpose**: Removes test studio_members and auth users created by seed-test-users.ts.
- **Usage**: `npm run db:rollback:users`
- **Warning**: Permanently deletes auth users (irreversible).
- **Operations**:
  1. Deletes 4 studio_member records by ID
  2. Deletes 4 auth users by email lookup
- **Calls To**: Deterministic member IDs (4 UUIDs)

### verify-test-users.ts
Verification script for test user integrity.
- **Purpose**: Validates that test users are correctly seeded.
- **Usage**: `npm run db:verify:users`
- **Checks**:
  1. Auth users exist for all 4 test emails
  2. studio_members records match expected IDs, roles, display names, studios
  3. Login test with `owner@test.com` / `Test@1234`
- **Exit Code**: 0 if all pass, 1 if any fail

### seed-ids.ts
Deterministic UUID constants for all seed entities.
- **Purpose**: Shared ID map ensuring consistency across seed scripts and tests.
- **Exports**: 80+ constant UUIDs for studios, users, members, packages, clients, leads, bookings, contracts, invoices, payments, assignments, galleries, face clusters, upload jobs, onboarding events, plans, and access tokens.
- **Token Pattern**: 64-char hex strings (e.g., `aaa...aaa`, `bbb...bbb`)
- **SEED_IDS Object**: Aggregated export of all IDs for bulk imports.
- **Side-effect free**: Safe to import from Vitest tests.

### tsconfig.seed.json
TypeScript configuration for seed scripts. Extends base tsconfig with seed-specific settings.
