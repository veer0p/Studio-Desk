## Purpose
Authentication guard middleware for Next.js API routes — validates JWT sessions and enforces role-based access (owner vs member).

## Key Functions

### `requireAuth(req)`
**File**: `guards.ts`
**Purpose**: Validates authenticated session, returns user + member context
**Parameters**: `{ req: NextRequest }`
**Returns**: `Promise<{ user, member, supabase }>` | throws `UNAUTHORIZED` or `FORBIDDEN`
**Async**: Yes

**Calls To** (Level 1):
- `createClient, createClientFromAccessToken` → `../supabase/.ai.md`
- `getMemberByUserId` → `../supabase/.ai.md`
- `Errors` → `../.ai.md`

**Called By** (Level 1):
- All authenticated API routes → `../../app/api/v1/.ai.md`
- `requireOwner()` → `./guards.ts`

### `requireOwner(req)`
**File**: `guards.ts`
**Purpose**: Stricter guard — only studio owners can proceed
**Parameters**: `{ req: NextRequest }`
**Returns**: `Promise<{ user, member, supabase }>` | throws `FORBIDDEN`
**Async**: Yes

**Calls To** (Level 1):
- `requireAuth()` → `./guards.ts`
- `Errors` → `../.ai.md`

**Called By** (Level 1):
- All write-operation API routes (POST/PATCH/DELETE) → `../../app/api/v1/.ai.md`

**Common Patterns**:
- Auth header detection: Bearer token → `createClientFromAccessToken()`, cookie → `createClient()`
- Uses `getUser()` (NOT `getSession()`) for security
- Member lookup via `getMemberByUserId()` with `is_active` filter
- `requireOwner()` wraps `requireAuth()` and adds role check
