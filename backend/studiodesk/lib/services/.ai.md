# Services Module

## Purpose
Core business logic layer for the StudioDesk backend. Contains 26 service classes/modules that orchestrate database operations, external integrations, validation, and business rules. All services use Supabase clients (standard or admin) for data access and delegate CRUD operations to repository modules.

## Architecture Pattern
- Services receive validated input, apply business rules, call repositories for DB operations
- External services (WhatsApp, Email, Razorpay, Immich) have dedicated client modules
- Fire-and-forget pattern used for non-critical side effects (audit logs, notifications)
- Error handling via centralized `Errors` factory (`@/lib/errors`)

## Key Services

### auth.service.ts
User authentication and profile management.
- **signup**: Creates auth user, studio, and owner member. Handles duplicate detection.
- **login**: Authenticates via Supabase, fetches member/studio info.
- **logout**: Signs out from Supabase (no-op if no session).
- **resetPassword**: Generates recovery link via Admin API.
- **updatePassword**: Updates user password.
- **updateMe**: Updates member profile (auth metadata + DB record).
- **me**: Returns current user, member, and studio info.
- **requestWhatsAppOtp / verifyWhatsAppOtp**: WhatsApp-based OTP authentication.
- **Async**: All methods are async.
- **Calls To**: `@/lib/repositories/studio.repo`, `@/lib/repositories/team.repo`, `@/lib/supabase/admin`, `@/lib/supabase/queries`, `@/lib/validations/auth.schema`
- **Called By**: API routes under `/api/v1/auth/**`

### admin-auth.service.ts
Platform admin authentication with session management and audit logging.
- **login**: Authenticates admin, verifies `platform_admins` record, creates session, logs audit.
- **logout**: Revokes admin session.
- **getAdminContext**: Gets current admin from session, validates active status.
- **getAdminByUserId**: Looks up platform admin by user_id.
- **createSession**: Creates admin session with 8-hour expiry.
- **validateSession**: Checks token, revocation, expiry (8h), idle timeout (30min).
- **updateSessionActivity**: Updates `last_active_at`.
- **updateLoginMetadata**: Updates `login_count` and `last_login_at`.
- **logAudit**: Fire-and-forget audit log insert.
- **sanitizeAdmin**: Removes `user_id` and `totp_secret` from response.
- **Async**: All methods are async.
- **Calls To**: `@/lib/supabase/admin`, `@/lib/validations/admin.schema`, `@/lib/errors`
- **Called By**: `@/lib/admin/guards.ts`, admin API routes

### two-fa.service.ts
TOTP-based two-factor authentication for platform admins.
- **generateSecret**: Creates base32 secret and OTP URI.
- **verify**: Verifies TOTP token with ±1 time-step window.
- **enable / disable**: Enables/disables 2FA on platform_admins record.
- **verifyForLogin**: Login-time verification wrapper.
- **isRequired**: Super admins always require 2FA; others depend on global setting.
- **Async**: `enable`/`disable` are async; others are sync.
- **Implementation**: HMAC-SHA1 per RFC 6238 (no external dependencies).
- **Calls To**: `@/lib/errors`, Node.js `crypto` module
- **Called By**: Admin 2FA API routes, `admin-auth.service.ts`
