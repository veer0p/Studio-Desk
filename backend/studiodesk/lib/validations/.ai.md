# Validations Module

## Purpose
Zod schema definitions for all API request validation across the StudioDesk backend. Provides type-safe input validation with TypeScript inference for all domain entities. Used by service classes to validate inputs before processing.

## Key Schemas

### auth.schema.ts
Authentication-related schemas.
- **signupSchema**: `email` (string, email), `password` (min 8), `fullName` (min 2), `studioName` (min 2), `studioSlug` (lowercase alphanumeric + hyphens).
- **loginSchema**: `email`, `password`.
- **resetPasswordSchema**: `email`.
- **updatePasswordSchema**: `password` (min 8).
- **updateMeSchema**: Partial update for `full_name`, `phone`, `whatsapp`, `preferred_language`, `designation`. Requires at least one field.
- **Types**: `SignupInput`, `LoginInput`, `ResetPasswordInput`, `UpdatePasswordInput`, `UpdateMeInput`
- **Calls To**: `./common.schema` (phoneSchema)
- **Called By**: `../services/auth.service.ts`

### admin.schema.ts
Platform admin authentication and management schemas.
- **adminLoginSchema**: `email`, `password` (required).
- **auditLogQuerySchema**: Paginated audit log query with optional filters (`admin_id`, `entity_type`, `action`).
- **studioListQuerySchema**: Studio listing with search, plan_tier, subscription_status, is_suspended filters.
- **createFeatureFlagSchema / updateFeatureFlagSchema**: Feature flag CRUD with tier targeting.
- **updatePlatformSettingSchema**: Platform setting value update.
- **suspendStudioSchema / impersonateStudioSchema**: Both require `reason` (min 10 chars).
- **createSupportNoteSchema**: `note` (max 5000), `is_flagged`.
- **setup2FASchema / verify2FASchema / enable2FASchema / login2FASchema**: TOTP 2FA flow schemas.
- **Types**: `CreateSupportNoteInput`, `Setup2FAInput`, `Verify2FAInput`, `Enable2FAInput`, `Login2FAInput`
- **Called By**: Admin API routes, `../services/admin-auth.service.ts`, `../services/admin-dashboard.service.ts`

### admin-plans.schema.ts
Subscription plan management schemas.
- **createPlanSchema**: `tier` (starter|studio|agency), `name`, `monthly_price_inr`, `annual_price_inr`, `max_team_members`, `max_bookings_per_month` (nullable), `storage_limit_gb`, Razorpay plan IDs, `features` array, `is_active`.
- **updatePlanSchema**: All fields optional.
- **Types**: `CreatePlanInput`, `UpdatePlanInput`
- **Called By**: `../services/admin-plan.service.ts`

### api-key.schema.ts
API key management schemas.
- **createApiKeySchema**: `name` (max 50), `scopes` (min 1), `expires_at` (optional datetime).
- **updateApiKeySchema**: Partial update.
- **Types**: `CreateApiKeyInput`, `UpdateApiKeyInput`
- **Called By**: API key management routes

### assignment.schema.ts
Team assignment and scheduling schemas.
- **assignmentRoleSchema**: Enum: photographer, videographer, assistant, editor, drone_operator, other.
- **assignmentStatusSchema**: Enum: pending, confirmed, declined.
- **assignmentInputSchema**: `member_id` (uuid), `role`, `notes` (max 500).
- **createAssignmentsSchema**: Array of 1-10 assignments.
- **updateAssignmentSchema**: Partial with cross-field validation (decline_reason required when status=declined).
- **teamScheduleQuerySchema**: Date range + optional member_id filter.
- **memberAssignmentsQuerySchema**: Date range + optional status filter.
- **shootBriefSchema**: Comprehensive shoot brief fields (times, venue, client info, shot list, equipment notes). Requires at least one field.
- **Types**: `AssignmentInput`, `UpdateAssignmentInput`, `TeamScheduleQueryInput`, `MemberAssignmentsQueryInput`, `ShootBriefInput`
- **Calls To**: `./common.schema` (dateSchema, phoneSchema, uuidSchema)
- **Called By**: `../services/assignment.service.ts`

### booking.schema.ts
Booking lifecycle schemas.
- **createBookingSchema**: Required: `title`, `client_id`, `event_type` (10 types), `total_amount`, `advance_amount`. Optional: `lead_id`, `event_date`, `package_id`, venue details, `notes`.
- **updateBookingSchema**: All fields optional, requires at least one field.
- **updateStatusSchema**: `status` enum (10 stages: new_lead through closed/lost).
- **Types**: `CreateBookingInput`, `UpdateBookingInput`
- **Event Types**: wedding, pre_wedding, engagement, portrait, birthday, corporate, product, maternity, newborn, other.
- **Called By**: `../services/booking.service.ts`

### client.schema.ts
Client CRM schemas.
- **createClientSchema**: Required: `full_name`, `phone`. Optional: `email`, `whatsapp`, address fields, `company_name`, `gstin`, `notes`, `tags` (max 10).
- **updateClientSchema**: Partial with at-least-one-field validation.
- **clientsQuerySchema**: Search + pagination.
- **Types**: `CreateClientInput`, `UpdateClientInput`, `ClientsQueryInput`
- **Calls To**: `./common.schema` (phoneSchema, pincodeSchema, gstinSchema)
- **Called By**: `../services/client.service.ts`

### common.schema.ts
Shared validation primitives used across all schemas.
- **amountSchema**: String matching `^\d{1,10}(\.\d{1,2})?$` (up to 10 digits + 2 decimals).
- **phoneSchema**: Indian mobile number `^[6-9]\d{9}$`.
- **gstinSchema**: Full GSTIN validation with checksum verification.
- **panSchema**: `^[A-Z]{5}[0-9]{4}[A-Z]{1}$`.
- **ifscSchema**: `^[A-Z]{4}0[A-Z0-9]{6}$`.
- **pincodeSchema**: `^[1-9][0-9]{5}$` (6-digit).
- **uuidSchema**: Standard UUID.
- **dateSchema**: `YYYY-MM-DD` format.
- **paginationSchema**: `page` (min 0, default 0), `pageSize` (1-100, default 20).
- **Calls To**: `@/lib/gstin-checksum` (isValidGstinChecksum)
- **Called By**: All other schema files in this directory

### contract.schema.ts
Contract and template schemas.
- **contractTokenSchema**: 64-char hex string.
- **createContractSchema**: `booking_id`, optional `template_id` or `custom_content`, optional `notes`.
- **updateContractSchema**: `content_html` (100-100000 chars), `notes`. Requires at least one field.
- **signContractSchema**: `signature_data` (max 50000 chars), optional `signed_name`.
- **createTemplateSchema / updateTemplateSchema**: Template name, event_type, content_html, is_default.
- **contractsQuerySchema**: Status filter + pagination.
- **Types**: `CreateContractInput`, `UpdateContractInput`, `SignContractInput`, `CreateTemplateInput`, `UpdateTemplateInput`, `ContractsQueryInput`
- **Calls To**: `./common.schema` (uuidSchema)
- **Called By**: `../services/contract.service.ts`

### gallery.schema.ts
Gallery management schemas.
- **galleriesQuerySchema**: Booking_id, status, pagination.
- **createGallerySchema**: `booking_id`, optional `name`.
- **updateGallerySchema**: Optional `name`, requires at least one field.
- **uploadPhotosSchema**: Array of 1-50 files with name, mimeType (image/video types), size (max 50MB), base64 data, optional taken_at.
- **uploadStatusQuerySchema**: `job_id` (uuid).
- **labelClusterSchema**: `label` (1-100 chars).
- **publishGallerySchema**: Optional `expires_at`, `allow_download` (default true).
- **selfieUploadSchema**: Single selfie image (max 5MB).
- **Types**: `GalleriesQueryInput`, `CreateGalleryInput`, `UpdateGalleryInput`, `UploadPhotosInput`, `PublishGalleryInput`, `SelfieUploadInput`
- **Calls To**: `./common.schema` (uuidSchema)
- **Called By**: `../services/gallery.service.ts`

### invoice.schema.ts
Invoice and payment schemas.
- **invoiceTokenSchema**: 64-char hex string.
- **invoiceLineItemSchema**: `sort_order`, `name`, `description`, `hsn_sac_code` (default 998389), `quantity` (default 1), `unit_price`.
- **createInvoiceSchema**: `booking_id`, `invoice_type` (advance|balance|full), optional `gst_type`, `hsn_sac_code`, `due_date`, `notes`, `internal_notes`, `line_items` (1-20).
- **updateInvoiceSchema**: Only `notes`, `internal_notes`, `due_date` editable.
- **recordPaymentSchema**: `amount`, `method` (8 types), optional `reference_number`, `payment_date`, `bank_name`, `notes`.
- **creditNoteSchema**: `amount`, `reason` (10-500 chars).
- **invoicesQuerySchema**: Status, type, booking_id, date range, pagination.
- **paymentsQuerySchema**: Invoice_id, booking_id, status, method, pagination.
- **financeSummaryQuerySchema**: `period` enum.
- **Types**: `CreateInvoiceInput`, `UpdateInvoiceInput`, `RecordPaymentInput`, `CreditNoteInput`, `InvoicesQueryInput`, `PaymentsQueryInput`, `FinanceSummaryQueryInput`
- **Calls To**: `./common.schema` (amountSchema, dateSchema, uuidSchema)
- **Called By**: `../services/invoice.service.ts`

### lead.schema.ts
Lead management schemas.
- **inquiryFormSchema**: Public inquiry form: `full_name`, `phone`, optional email, event_type, event_date, venue, budget range, message, guest_count.
- **createLeadSchema**: Manual lead creation with client_id or full contact details, source (8 types), priority (3 levels).
- **updateLeadSchema**: Status, priority, date, venue, budget, follow_up_at, notes, assigned_to. Requires at least one field.
- **convertLeadSchema**: Converts to booking: optional title, required event_date, optional amounts, package_id, gst_type.
- **leadsQuerySchema**: Filters by status, source, event_type, assigned_to, search, date range, pagination.
- **Types**: `InquiryFormInput`, `CreateLeadInput`, `UpdateLeadInput`, `ConvertLeadInput`, `LeadsQueryInput`
- **Calls To**: `./common.schema` (phoneSchema, uuidSchema, dateSchema, amountSchema)
- **Called By**: `../services/lead.service.ts`

### package.schema.ts
Service package and addon schemas.
- **lineItemSchema**: `name`, `description`, `hsn_sac_code`, `quantity` (default 1), `unit_price`.
- **createPackageSchema**: `name`, `event_type` (10 types), optional `description`, `base_price`, `hsn_sac_code` (default 998389), `is_gst_applicable` (default true), `deliverables` (max 20), `turnaround_days` (1-365), `line_items` (max 20).
- **updatePackageSchema**: Partial with at-least-one-field validation.
- **createAddonSchema / updateAddonSchema**: `name`, `description`, `price`, `unit` (flat|per_hour|per_day).
- **Types**: `CreatePackageInput`, `UpdatePackageInput`, `CreateAddonInput`, `UpdateAddonInput`
- **Calls To**: `./common.schema` (amountSchema)
- **Called By**: `../services/package.service.ts`

### proposal.schema.ts
Proposal management schemas.
- **listProposalsSchema**: Pagination + status/booking_id filters.
- **proposalLineItemSchema**: Full line item with `item_type`, `sort_order`, `addon_id`.
- **createProposalSchema**: `booking_id`, `client_id`, `gst_type` (none|cgst_sgst|igst), optional `valid_until`, `notes`, `line_items` (min 1).
- **updateProposalSchema**: Partial with strict mode, line_items optional.
- **acceptProposalSchema**: `action` (accept|reject), optional `reason`.
- **Types**: `ListProposalsInput`, `CreateProposalInput`, `UpdateProposalInput`, `AcceptProposalInput`, `ProposalLineItemInput`
- **Calls To**: `./common.schema` (uuidSchema, dateSchema, paginationSchema)
- **Called By**: `../services/proposal.service.ts`

### referral.schema.ts
Referral program schemas.
- **redeemReferralSchema**: `code`, `new_studio_id` (uuid).
- **Types**: `RedeemReferralInput`
- **Called By**: Referral API routes

### settings.schema.ts
Studio settings schemas.
- **updateNotificationSchema**: Boolean flags for notification triggers, working hours (HH:MM format), timezone. Requires at least one field.
- **updateIntegrationsSchema**: WhatsApp provider/type, Razorpay account ID. Immiсh fields commented out (TODO). Requires at least one field.
- **testIntegrationSchema**: `service` (whatsapp|immich|razorpay), `test_phone` required for WhatsApp.
- **Types**: `UpdateNotificationInput`, `UpdateIntegrationsInput`, `TestIntegrationInput`
- **Calls To**: `./common.schema` (phoneSchema)
- **Called By**: `../services/settings.service.ts`

### studio.schema.ts
Studio profile and onboarding schemas.
- **step1Schema** (Basic Info): `name`, optional `phone`, `city`, `state`, optional `specialty`, `experience`, `tagline`.
- **step2Schema** (Business Details): Optional `gstin`, `pan`, `business_address`, `pincode`, `email`, `website`.
- **step3Schema** (Payment Setup): Optional `bank_name`, `bank_account_number`, `bank_ifsc`, `invoice_prefix`, `default_advance_pct`.
- **step4Schema** (Inquiry Form): Optional form customization booleans and text fields.
- **step5Schema** (First Package): Optional `package` object with name, event_type, base_price, deliverables, turnaround_days.
- **completeStepSchema**: `data` (record), optional `time_spent_sec` (0-86400).
- **STEP_SCHEMAS / STEP_NAMES / STEP_LABELS**: Mappings for 5 onboarding steps.
- **updateProfileSchema**: Comprehensive studio profile update with strict validation (GSTIN, PAN, IFSC, pincode, phone, email, brand color hex, slug, etc.).
- **Types**: `Step1Input`, `Step5Input`, `CompleteStepInput`, `UpdateProfileInput`
- **Constants**: `STEP_NAMES`, `STEP_LABELS`, `STEP_SCHEMAS`
- **Calls To**: `./common.schema` (gstinSchema, panSchema, pincodeSchema, phoneSchema, ifscSchema, amountSchema)
- **Called By**: `../services/studio.service.ts`

### team.schema.ts
Team management schemas.
- **inviteMemberSchema**: `email` (valid), `role` (photographer|videographer|editor|assistant).
- **updateRoleSchema**: Same role enum.
- **tokenSchema**: 64-char hex string.
- **Types**: `InviteMemberInput`, `UpdateRoleInput`
- **Called By**: `../services/team.service.ts`

### automation.schema.ts
Automation configuration schemas.
- **automationTypes**: 15 automation type constants (lead_acknowledgment through post_shoot_followup).
- **automationTypeSchema**: Enum from automationTypes.
- **automationChannelSchema**: whatsapp|email|both.
- **automationStatusSchema**: pending|sent|failed|cancelled.
- **automationPeriodSchema**: today|this_week|this_month|last_month.
- **updateSettingSchema**: Single setting update with delay_days (0-30), delay_hours (0-23), send_time (HH:MM).
- **updateSettingsSchema**: Array of 1-15 settings.
- **triggerManualSchema**: Requires booking_id OR lead_id.
- **testMessageSchema**: automation_type, phone, optional channel.
- **automationLogQuerySchema**: Comprehensive log query with filters.
- **statsQuerySchema**: Period filter.
- **Types**: `UpdateSettingInput`, `UpdateSettingsInput`, `TriggerManualInput`, `TestMessageInput`, `AutomationLogQueryInput`, `StatsQueryInput`
- **Calls To**: `./common.schema` (dateSchema, phoneSchema, uuidSchema)
- **Called By**: `../services/automation.service.ts`
