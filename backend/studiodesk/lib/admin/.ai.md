## Purpose
Platform admin authentication guards — validates Supabase auth and verifies `platform_admins` record.

## Key Functions

### `requireAdminAuth(req)`
**File**: `guards.ts`
**Purpose**: Require authenticated platform admin (Supabase auth + platform_admins check)
**Parameters**: `{ req: NextRequest }`
**Returns**: `Promise<{ admin, user, supabase }>` | throws `UNAUTHORIZED` or `FORBIDDEN`
**Async**: Yes

**Calls To** (Level 1):
- `createClient, createClientFromAccessToken` → `../supabase/.ai.md`
- `AdminAuthService.getAdminByUserId()` → `../services/.ai.md`
- `Errors` → `../.ai.md`

**Called By** (Level 1):
- `POST /api/v1/admin/auth/logout` → `../../app/api/v1/.ai.md`
- `GET /api/v1/admin/auth/me` → `../../app/api/v1/.ai.md`
- All future admin API routes

### `requireSuperAdmin(req)`
**File**: `guards.ts`
**Purpose**: Stricter guard — only `super_admin` role can proceed
**Parameters**: `{ req: NextRequest }`
**Returns**: `Promise<{ admin, user, supabase }>` | throws
**Async**: Yes

**Calls To** (Level 1):
- `requireAdminAuth()` → `./guards.ts`
- `Errors` → `../.ai.md`

**Common Patterns**:
- Admins authenticate via Supabase Auth (same as studio users)
- After auth, `platform_admins` table is checked for matching `user_id`
- Session tokens stored in `admin_sessions` table with 8-hour expiry
- All actions logged to `admin_audit_logs` (append-only)
