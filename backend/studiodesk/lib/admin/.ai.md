# Admin Module

## Purpose
Authentication and authorization guards for the StudioDesk platform admin panel. Provides middleware functions that validate platform admin sessions and enforce role-based access control (RBAC).

## Key Files

### guards.ts
Express/Next.js middleware guards for admin routes.

**Functions:**

#### requireAdminAuth(req: NextRequest) -> Promise<AdminGuardResult>
Authenticates platform admin requests.
- **Parameters**: `req` (NextRequest) - the incoming HTTP request
- **Returns**: `AdminGuardResult` object with `admin`, `user`, and `supabase` properties
- **Async**: Yes
- **Logic**:
  1. Extracts session token from `Authorization: Bearer <token>` header OR `admin_session` cookie
  2. Creates admin Supabase client (service role, bypasses RLS)
  3. Validates session token via `AdminAuthService.validateSession()` (checks existence, revocation, 8h expiry, 30min idle timeout)
  4. Verifies admin is active
  5. Updates session `last_active_at`
  6. Returns admin context
- **Errors**: Throws `Errors.unauthorized()` if no token or invalid session

#### requireSuperAdmin(req: NextRequest) -> Promise<AdminGuardResult>
Requires `super_admin` role specifically.
- **Parameters**: `req` (NextRequest)
- **Returns**: `AdminGuardResult`
- **Async**: Yes
- **Logic**: Calls `requireAdminAuth()`, then checks `admin.role === 'super_admin'`
- **Errors**: Throws `Errors.unauthorized()` if auth fails, `Errors.validation('Super admin access required')` if role check fails

**Interfaces:**

```typescript
interface AdminGuardResult {
  admin: any        // platform_admins record (sanitized)
  user: any         // { id, email } from Supabase auth
  supabase: any     // Supabase admin client
}
```
