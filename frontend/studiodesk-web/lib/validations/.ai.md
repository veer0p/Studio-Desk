## Purpose
Client-side Zod validation schemas for form inputs across the frontend.

## Key Schemas

### Booking (`booking.ts`)
| Schema | Purpose | Key Fields |
|---|---|---|
| `bookingSchema` | Full booking form | 14 fields: `client_id`, `title`, `event_type` (10-value enum), `event_date`, `call_time`, `venue_name`, `venue_city`, `venue_address`, `package_id`, `notes`, `status` (9-value enum), `total_amount` |
| `createBookingSchema` | Create validation | `bookingSchema` with refinement requiring `event_date`, `title`, `client_id` |
| `updateBookingSchema` | Update validation | `.partial()` of `bookingSchema` |

**Types Exported**: `CreateBookingData`, `UpdateBookingData`, `BookingFormData`

**Called By** (Level 1):
- Booking form dialogs → `../../components/.ai.md`

### Client (`client.ts`)
| Schema | Purpose | Key Fields |
|---|---|---|
| `newClientSchema` | Create client | 9 fields: `fullName`, `phone` (min 10), `whatsapp?`, `email?`, `city?`, `source?`, `referredBy?`, `tags?`, `notes?` |
| `editClientSchema` | Edit client | Extends `newClientSchema` with `dateOfBirth?`, `anniversary?` |

**Types Exported**: `NewClientData`, `EditClientData`

**Called By** (Level 1):
- `NewClientDialog`, `EditClientSheet` → `../../components/.ai.md`

### Invoice (`invoice.ts`)
| Schema | Purpose | Key Fields |
|---|---|---|
| `lineItemSchema` | Single line item | `description`, `amount` (regex validated), `gst_rate?`, `quantity` (coerced number, min 1) |
| `invoiceSchema` | Full invoice | 8 fields: `client_id`, `booking_id?`, `issue_date`, `due_date`, `type` (advance/balance/full/credit_note), `line_items[]` (min 1), `notes?`, `gst_type?` |
| `createInvoiceSchema` | Create alias | `invoiceSchema` |
| `updateInvoiceSchema` | Update | `.partial()` of `invoiceSchema` |

**Types Exported**: `CreateInvoiceData`, `UpdateInvoiceData`, `InvoiceLineItem`

**Called By** (Level 1):
- `CreateInvoiceDialog` → `../../components/.ai.md`
