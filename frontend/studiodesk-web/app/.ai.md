## Purpose
Next.js App Router defining all pages and layouts. Route groups define auth boundaries (auth, dashboard, portal, public).

## Route Structure

### Auth Routes `(auth)/` — No auth required
| Route | Component | Main Component | Data Source |
|---|---|---|---|
| `(auth)/login` | Client | `LoginForm` | `signIn()` → `../lib/.ai.md` |
| `(auth)/signup` | Client | `SignupForm` | `signUp()` → `../lib/.ai.md` |
| `(auth)/forgot-password` | Client | `ForgotPasswordForm` | `forgotPassword()` → `../lib/.ai.md` |

### Dashboard Routes `(dashboard)/` — Protected
| Route | Component | Main Component | Data Source |
|---|---|---|---|
| `(dashboard)/dashboard` | Server | `DashboardShell` | SWR → `fetchDashboardOverview()` |
| `(dashboard)/bookings` | Server | `BookingsList` / `KanbanBoard` | SWR → `fetchBookingsList()` |
| `(dashboard)/bookings/[id]` | Client | `BookingSlideOver` | SWR → `fetchBookingDetail()` |
| `(dashboard)/clients` | Server | `ClientsShell` → `ClientsGrid`/`Table` | SWR → `fetchClientsList()` |
| `(dashboard)/clients/[id]` | Client | `ClientDetailPage` | SWR → `fetchClientDetail()` |
| `(dashboard)/leads` | Server | `LeadsShell` → `LeadsKanban` | SWR fetcher |
| `(dashboard)/proposals` | Server | `ProposalShell` → `ProposalList` | SWR → `fetchProposalsList()` |
| `(dashboard)/contracts` | Server | `ContractsShell` → `ContractList` | SWR → `fetchContractsList()` |
| `(dashboard)/finance` | Server | `FinanceShell` → `FinanceSummaryBar` | SWR → `fetchFinanceSummary()` |
| `(dashboard)/finance/[id]` | Client | `InvoiceDetail` | SWR → `fetchInvoiceDetail()` |
| `(dashboard)/gallery` | Server | Gallery list/grid | SWR → `fetchGalleriesList()` |
| `(dashboard)/gallery/[id]` | Client | `GalleryDetail` | SWR → `fetchGalleryDetail()` |
| `(dashboard)/team` | Server | `TeamShell` → `TeamMemberCard` | SWR → `fetchTeamMembers()` |
| `(dashboard)/team/[id]` | Client | Team member detail | Props |
| `(dashboard)/addons` | Server | `AddonsShell` → `AddonList` | SWR → `fetchAddonsList()` |
| `(dashboard)/analytics` | Server | `AnalyticsShell` → period tabs | SWR → `fetchAnalytics*()` |
| `(dashboard)/settings/*` | Client | Settings tabs (owner, studio, packages, finance, integrations, notifications, billing, danger) | Various SWR |

### Auth Reset (`auth/`)
| Route | Component | Purpose |
|---|---|---|

## Data Fetching Pattern
- **Server Components**: Fetch from backend API URLs, pass as props to shells
- **Client Components**: `useSWR(fetcher)` with deduping from `../lib/.ai.md`
- **List → Detail**: Detail panels fetch own data; list remains stable (no re-render on URL param change)

## Auth Pattern
- `(auth)/` → No auth, redirect to dashboard if logged in
- `(dashboard)/` → Protected, redirect to login if unauthenticated
- `portal/` → Token-based auth via `PortalAuthProvider` → `../lib/.ai.md`
- `gallery/`, `inquiry/` → Public
- `onboarding/` → Protected, post-signup flow
