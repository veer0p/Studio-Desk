**Language**: TypeScript/React (Next.js App Router)

## Purpose
Admin route group — platform administration panel with dark theme, separate from studio dashboard.

## Route Structure

| Route | Component | Purpose |
|---|---|---|
| `(admin)/layout.tsx` | Server | Dark theme root layout (`bg-zinc-950`) |
| `(admin)/admin/layout.tsx` | Server | Admin layout with sidebar |
| `(admin)/admin/login` | Client | Admin login page (email + password) |
| `(admin)/admin/dashboard` | Client | Platform overview KPIs + health alerts |
| `(admin)/admin/studios` | Client | Studio list with search, filters, pagination |
| `(admin)/admin/studios/[id]` | Client | Studio detail with suspend/reactivate/impersonate |
| `(admin)/admin/plans` | Client | Subscription plan cards with pricing |
| `(admin)/admin/audit-logs` | Client | Paginated audit log viewer with filters |
| `(admin)/admin/feature-flags` | Client | Feature flag management (create, toggle, delete) |
| `(admin)/admin/settings` | Client | Platform settings overview |

## Data Fetching
- **Auth**: `useAdminAuth()` hook → SWR → `getAdminMe()` → `GET /api/v1/admin/auth/me`
- **Dashboard**: SWR → `fetchAdminDashboard()` → `GET /api/v1/admin/dashboard`
- **Studios**: SWR → `fetchAdminStudios()` / `fetchAdminStudioDetail()` → backend admin routes
- **Actions**: Direct fetch mutations → `suspendStudio()`, `reactivateStudio()`, `impersonateStudio()`

## Called By
- Admin users navigating to `/admin/*`
