# Leads Module

- **Path:** `components/leads/`
- **Last Updated:** 2026-04-13
- **Language:** TypeScript / React (Next.js)

## Purpose

Manages pre-booking inquiries through a kanban or list view. Leads represent potential bookings in Inquiry, Proposal Sent, or Negotiation stages. Users can search, filter, create leads, and mark them as Qualified (Won).

## Key Components

| Component | File | Purpose | Props | Client/Server |
|-----------|------|---------|-------|---------------|
| LeadsClient | `LeadsClient.tsx` | Main client wrapper. Fetches leads via SWR, handles search debouncing (300ms), toggles kanban/list views based on URL `?view=` param. | `search: string, view: string` | Client |
| LeadsShell | `LeadsShell.tsx` | Layout shell with header, search input, filter dropdown, view toggle (kanban/list), and Quick Add button. | `children, count, searchQuery, onSearchChange, filterOpen, onFilterOpenChange, overdueCount` | Client |
| LeadsKanban | `LeadsKanban.tsx` | Kanban board with 4 stages (Inquiry, Proposal Sent, Negotiation, Won). Responsive: vertical stacked on mobile, horizontal columns on desktop. | `labs: BookingSummary[]` | Client |
| LeadsList | `LeadsList.tsx` | TanStack Table-based list view with sorting, pagination (25/page), and row actions (view details, mark qualified, convert to booking). | None (fetches own data) | Client |
| LeadCard | `LeadCard.tsx` | Individual kanban card showing client name, event type, date, venue, amount. Links to `/leads/:id`. | `lead: BookingSummary` | Client |
| NewLeadDialog | `NewLeadDialog.tsx` | Dialog form to create a new lead. Validates Indian mobile numbers. Calls `createBooking()` with stage "Inquiry". Invalidates SWR cache on success. | `children?: React.ReactNode` | Client |

## Data Fetching

### SWR Hooks
- **LeadsClient / LeadsList:** `useSWR('/api/v1/bookings?stage=Inquiry,Proposal%20Sent,Negotiation&search=${searchQuery}', fetchBookingsList)`
- Both components fetch the same endpoint; LeadsList has its own independent fetch.

### Mutations
- `createBooking(bookingData)` — called by `NewLeadDialog` on form submit
- `updateBookingStage(id, "Won")` — called by `LeadsList` row action "Mark qualified"
- Both mutations call `mutate(key => key.startsWith('/api/v1/bookings'))` to revalidate

### API Functions Used
- `fetchBookingsList` — GET `/api/v1/bookings`
- `createBooking` — POST `/api/v1/bookings`
- `updateBookingStage` — PATCH booking stage

## UX Patterns

- **Search debouncing:** 300ms delay before URL param update via `router.replace()` with `{ scroll: false }`
- **View toggle:** URL-driven `?view=kanban|list` — switching does not refetch data
- **Kanban responsive:** Mobile shows stacked columns; desktop shows horizontal scrollable columns with `w-72` fixed width
- **Empty states:** Dashed border placeholders in kanban, centered message in list
- **Loading state:** Minimal monospace "Loading leads..." text with pulse animation
- **Optimistic revalidation:** `mutate()` on global booking keys after mutations
